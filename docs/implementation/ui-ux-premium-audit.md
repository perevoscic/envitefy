# Envitefy Premium UI/UX Audit

Date: 2026-06-04

## Audit Scope

This audit maps the major route groups and sampled implementation surfaces in the current app. The highest-risk product surfaces are the authenticated dashboard, Concierge V2 event workspace, event creation/customization routes, public guest event page, smart signup, and admin tooling.

## Cross-Cutting Problems

- Product navigation is feature-based, not journey-based. The user path should be Create with Concierge, Review draft, Publish guest page, Share, Track RSVPs, Manage schedule/forms/signups/payments/reminders, Send updates.
- Several screens use object words from the data model: resource, occurrence, source document, extracted item, config. These should become setup item, schedule item, uploaded flyer or file, found detail, and settings.
- Mode-aware language exists in pockets but not consistently. Birthday/social, school/class, gymnastics/team, and business events need distinct labels.
- Empty states often state absence rather than purpose and next action.
- Top metric cards often show zeros without explaining what the user should do next.
- Workspace pages duplicate local primitives rather than using shared PageHeader, cards, status chips, tabs, empty states, and error/loading components.
- Mobile is mostly responsive grids, but forms and side panels need more deliberate stacked flows and sticky primary actions.

## Priority Implementation Decisions

1. Build shared premium workspace primitives under `src/components/ui/premium-shell.tsx`.
2. Add centralized event-mode copy and resource/setup taxonomy under `src/lib/concierge-v2/mode-copy.ts`.
3. Refactor Concierge V2 Resources first because it contains the clearest product-language bug.
4. Apply the same header/card/empty/error patterns to Schedule, RSVP, Imports, Hub, and Concierge creation where feasible.
5. Preserve the public guest page renderer behavior and owner tools separation.
